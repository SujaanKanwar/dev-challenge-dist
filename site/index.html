Description

  • Data Emitter : Will emit the latest data received. Anyone listening to 'data-receive' event will get the latest data.
  • Table Module: This will initialize and manage data emitter and html renderer.This will get the data from Data emitter and will parse it to html renderer.
  • HTML renderer: This will be initialized by table module. It will get data from table module and render results to UI.

Design To Consider

  • All the dependencies of table module have been injected to it while initializing.
  • Observable design pattern is being used to emit the data received by data emitter.

Which libraries/frameworks ?

  • Whole table is getting re-render, for even a single row change.
    Solution: Library/framework which uses Virtual DOM to update the UI will be best suited as only diff should be updated in the UI.
    React will be a good choice, because it uses VDOM and it has good community support.

  • Maintenance, could become issue when this application will grow.
    Solution: Library/framework which store the state of the application, so that time traversal is possible while debugging. Also a single source of truth for state will help in developing further.
    Redux/Flux, will be a good choice in this case, which is designed for managing application state.
